## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner -> repo files | The new pytest reads repo-local README.md, docs/adoption.md, and skill dirs. No untrusted input crosses this boundary. |

No runtime attack surface: docs (README, docs/adoption.md) are static markdown; the only executable
artifact is a read-only pytest that enumerates repo-local `*/skills/*/SKILL.md` files and greps two
committed docs. There is no network, auth, secret handling, deserialization, or user-supplied input.
